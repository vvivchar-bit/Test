# Тестування вимог програмної системи «Готель»

## Перелік вимог

| ID | Назва в ReQtest | Зміст вимоги українською | Пріоритет |
|---|---|---|---|
| FR-01 | View rooms | Система відображає список номерів готелю з типом, ціною, описом і статусом доступності. | Must |
| FR-02 | View room details | Користувач може переглядати детальну інформацію про обраний номер. | Should |
| FR-03 | Create booking request | Клієнт може створити заявку на бронювання, заповнивши форму. | Must |
| FR-04 | Validate input data | Система перевіряє обов’язкові поля форми бронювання. | Must |
| FR-05 | Save booking request | Після успішного заповнення форми заявка зберігається у списку бронювань. | Must |
| FR-06 | View booking list | Адміністратор або користувач може переглянути список створених бронювань. | Should |
| FR-07 | Change booking status | Адміністратор може змінювати статус заявки на бронювання. | Should |
| FR-08 | Edit room information | Адміністратор може редагувати інформацію про номери. | Should |
| FR-09 | Search and filter rooms | Користувач може виконувати пошук і фільтрацію номерів. | Should |
| NFR-01 | Usability | Інтерфейс системи є зрозумілим і простим у використанні. | Should |
| NFR-07 | Browser compatibility | Система коректно відкривається і працює в сучасному веббраузері. | Should |

## Результат

Основні вимоги були перевірені за допомогою тест-кейсів у ReQtest. Адміністративні вимоги FR-07 і FR-08 залишені для подальшого тестування після повної реалізації адміністративного модуля.
